Rules

How to write rules

One rule per line. By default, matching parameters and everything after them are removed. Add ~ to remove only that single parameter.

Prefixes: Use ?, &, or # to match specific types. Omit the prefix to match all three (?param, &param, #param).

  • param or param~ - Global rule (all sites)
  • domain.com:param or domain.com:param~ - Domain-specific rule
  • !param - Never remove from any domain
  • !domain.com:param - Never remove from this domain
  • !domain.com - Whitelist domain (never clean)
  • param* - Wildcard matching (e.g., utm_source* matches utm_source_extra)

Lines starting with // are comments.
